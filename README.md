# DSA-Practice Repository

Welcome to the **DSA-Practice** repository! This repo is designed for **daily practice of data structures and algorithms in Python**, following a structured 1-month plan.

---

## **Repository Structure**

```
DSA-Practice/
│
├─ README.md                  # Overview of the repo and plan
├─ .gitignore
├─ Arrays/                    # Array problems
├─ Strings/                   # String manipulation problems
├─ Recursion/                 # Recursion problems
├─ LinkedLists/               # Linked list problems
├─ Stack/                     # Stack problems
├─ Queue/                     # Queue problems
├─ Searching/                 # Searching algorithms
├─ Sorting/                   # Sorting algorithms
├─ Trees/                     # Tree problems
├─ Graphs/                    # Graph problems
├─ DynamicProgramming/        # DP problems
├─ Misc/                      # Misc/combined concept problems
└─ utils/                     # Helper functions
```

Each folder contains problems related to that **data structure or algorithm**, named with `Day_X_ProblemName.py` to maintain chronological order.

## **Git Commit Guidelines**

* Commit **each problem separately** for clarity.
* Use the following **commit message format**:

```
feat([Topic]): solved [Problem Name] using [Approach/Algorithm]
```

**Examples:**

```
git commit -m "feat(Arrays): solved max/min array problem using iteration"
git commit -m "feat(Stack): solved valid parentheses problem using stack"
git commit -m "feat(DynamicProgramming): implemented coin change problem using bottom-up DP"
```
---

## **Tips for Effective Practice**

1. Solve **all problems in Python** with clean, readable code.
2. For hard problems, first write a **brute-force solution**, then optimize.
3. Use **small test cases** to debug and validate your solutions.
4. Maintain a **progress journal** (`progress.md`) to track solved problems.
5. Optional: Use LeetCode/HackerRank to cross-check solutions.
